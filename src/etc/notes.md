# Changelog & Design Notes

## Изменения

### 1. Рефакторинг: разбиение монолитного main.cpp на модули
- Исходный файл (1247 строк) разделён на логические модули:
  - `imp/` — точка входа, игровая логика, ввод
  - `sqd/` — генерация карты, рендеринг
  - `cfg/` — загрузка конфигурации
  - `types.h` — общие типы и объявления
- Принцип: каждая единица компиляции отвечает за одну предметную область.

### 2. Конфигурация (rts.cfg + cfg/config.cpp)
- INI-подобный формат, парсится в `GameConfig`.
- Параметры: `primary_color`, `secondary_color`, `update_rate`, `author_menu_button`, `tip_N`.
- `color_name()` / `parse_color()` для двустороннего преобразования цветов.
- `save_config()` — запись изменений обратно в файл (используется из меню Settings).

### 3. Command mode (`:`)
- Vim-подобный режим: `:attack <type>`, `:economy`, `:help`, `:menu`.
- `getnstr()` для безопасного ввода строки, отдельная функция `handle_command()`.

### 4. Меню (Main Menu, Settings, Plugins, About)
- Главное меню при старте игры (`:menu` возвращает в него).
- Навигация стрелками, выбор Enter, выход Q.
- **Settings**: редактирование cfg с подсказками; после сохранения перезагружаются цвета.
- **Plugins**: список плагинов, включение/выключение (Space), запуск (Enter), добавление кастомных (A).
- **About**: текст из `author_menu_button` в cfg.

### 5. Plugin system (Lua 5.4)
- `runner.lua` загружает плагин через `dofile()` и вызывает `plugin.run()`.
- Встроенные плагины:
  - Logger — пишет лог в logs/game.log
  - Calculator — математические выражения
  - Translator — EN→ES/RU словарь
  - Template — шаблон для разработки
  - Screenshot — дамп экрана в файл
  - Cleaner — подсчёт строк проекта + удаление .o
- Кастомные плагины добавляются через меню (указание пути).
- Плагины хранятся в `etc/lua/`.

### 6. Пиксельный шейдер в главном меню
- Анимированные цветные пиксели в фоне меню.
- Использует `menu_page` как счётчик кадров анимации.
- Разные символы (`.`, `*`, `+`, `#`) и цвета (CP_C1, CP_C2, CP_UI, CP_SEL) для разнообразия.

### 7. Структура проекта
```
rts/
├── imp/        — main.cpp, game.cpp, input.cpp
├── sqd/        — mapgen.cpp, render.cpp
├── cfg/        — config.cpp
├── etc/lua/    — Lua 5.4 плагины + runner.lua
├── types.h     — общие типы
├── rts.cfg     — конфигурация
└── Makefile    — сборка (g++, ncursesw, lua5.4)
```

## Чем руководствовался

- **Модульность**: каждый файл — одна ответственность (Single Responsibility).
- **Минимум зависимостей**: только libncursesw и liblua5.4, остальное — plain C++17 и Lua 5.1/5.4.
- **Расширяемость**: plugin system позволяет добавлять Lua-скрипты без перекомпиляции C++.
- **User-friendly**: меню с подсказками вместо ручного редактирования cfg.
- **Совместимость**: Lua код проверен на 5.1 и 5.4 (замена `loadstring` → `load` с fallback, замена `goto` на `else`).
- **Производительность**: ncurses timeout 100ms, карта 500×500 генерируется процедурно.
